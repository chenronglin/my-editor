# 小说编辑器使用与业务需求指南

这是一个精简版的小说编辑器，专门针对编辑和作者的协作场景设计。系统不仅支持划词批注，还提供了修订追踪与仅编辑可用的“编辑建议”功能。

---

## 1. 角色身份与协作权限

系统分为 **编辑（编辑A）** 和 **作者（作者B）** 两个用户角色。

### 开启与关闭修订的锁定规则：
- **编辑 开启修订**：
  - 编辑可以修改文档内容，此时**正文被锁定，作者端会自动进入只读状态**，无法修改正文。
- **编辑 结束修订（或未开启修订时）**：
  - 此时**作者可以修改文档内容，而编辑无法修改正文**。
  - **无缝修订行为一致性**：只要编辑没有开启修订，作者在工作模式中的编辑行为与编辑开启修订后是一致的，所做的任何增加、删除、替换等修改，都会被系统完整追踪。

---

## 2. 内容修改与修订追踪机制

系统提供轻量且清晰的修改追踪，**不需要“接受、确认”等复杂的审核流程**，只需要明确标注是谁修改的以及修改的时间。

- **工作模式**：
  - 用户可在此模式下修改文档并查看所有修订记录。
  - 修订记录包括**增加（浅绿底纹加绿色下划线）**、**删除（浅红底纹加红色删除线）**和**替换（删除原有字符后输入新字符）**。
  - **修订标签**：每个修订记录之前都会展示一个形如 `[修订人 修改时间]` 的防篡改标签（例如 `[编辑A 05-28 12:35]`），精准记录是谁在何时做出的变动。
- **阅读模式（Clean 版本）**：
  - 用于预览小说的最终效果。
  - 所有的删除修订彻底隐藏，新增的修订去掉绿字和下划线，作为普通正文渲染。
  - 所有的修订人与时间标签隐藏。
  - 正文的所有黄色批注高亮底纹隐藏，且编辑建议卡片隐藏。

---

## 3. 批注与回复（不修改正文）

用户针对选定的正文内容进行评价，不修改正文内容。

1. **新增批注**：在正文中用鼠标选中一段文字，在弹出的浮动工具栏中点击“添加批注”，在右侧输入内容即可。被批注的内容在工作模式下呈**黄色高亮**。
2. **回复批注**：编辑与作者均可以在右侧批注区针对对应的讨论卡片进行回复，形成讨论串。
3. **删除批注**：完成讨论后，可以点击垃圾桶图标删除单条批注或整个讨论串。

---

## 4. 编辑建议（卡片化建议）

编辑可以在任何段落后面插入一段“编辑建议”（类似黄色背景的卡片框）。

- **使用场景**：在工作模式下，供编辑撰写多条大纲性或逻辑性的调整方案，但不作为正文文字。
- **操作权限**：
  - **只允许编辑操作**：编辑角色在工作模式下，可点击工具栏的 **【📝 添加建议】** 按钮在当前段落后插入卡片。编辑可在卡片内自由输入/换行、或点击“删除建议”销毁卡片。
  - **作者只读**：作者角色对此卡片仅能阅读，无法对其进行任何文字修改。
  - **不可评论**：卡片属性设置为不可编辑（`contenteditable="false"`），无法被主编辑器的划词功能选择，因此**不能对其添加批注评论**。
- **最终阅读模式表现**：
  - 不进入最终版本，在阅读模式下该建议卡片**完全不可见（自动隐藏）**。
